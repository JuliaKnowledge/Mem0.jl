Providers API

Abstract interfaces and concrete implementations for LLM, embedding, vector store, and graph store providers.

Abstract Types

Mem0.AbstractLLM — Type
AbstractLLM

Abstract base type for language model providers.

Mem0.AbstractEmbedder — Type
AbstractEmbedder

Abstract base type for embedding model providers.

Mem0.AbstractVectorStore — Type
AbstractVectorStore

Abstract base type for vector store backends.

Mem0.AbstractGraphStore — Type
AbstractGraphStore

Abstract base type for graph memory backends. All concrete subtypes must have llm::AbstractLLM and custom_prompt::Union{Nothing, String} fields.

LLM Providers

OpenAILLM <: AbstractLLM

OpenAI-compatible LLM provider using the Chat Completions API.

Mem0.OllamaLLM — Type
OllamaLLM <: AbstractLLM

Ollama-compatible LLM provider using the local Ollama API.

Embedding Providers

OpenAI embedding provider using the Embeddings API.

Mem0.OllamaEmbedding — Type
OllamaEmbedding <: AbstractEmbedder

Ollama embedding provider using the local Ollama API.

Vector Store Providers

Mem0.InMemoryVectorStore — Type
InMemoryVectorStore <: AbstractVectorStore

A simple in-memory vector store using brute-force cosine similarity search. Suitable for development, testing, and small-scale deployments.

Graph Store Providers

Mem0.InMemoryGraphStore — Type
InMemoryGraphStore <: AbstractGraphStore

A simple in-memory graph store using adjacency lists. Supports entity embeddings for semantic search, soft-deletion, and multi-tenant filtering.

Mem0.Neo4jGraphStore — Type
Neo4jGraphStore <: AbstractGraphStore

Graph store backed by Neo4j, accessed via the HTTP Transactional Cypher API. Stores entity embeddings as float-array node properties for client-side cosine similarity matching, and uses Cypher MERGE for upsert semantics.

Configuration

Pass a config dict with keys:

  • "url" — Neo4j HTTP API URL (default "http://localhost:7474") Also accepts bolt/neo4j URLs (neo4j://host:port) — the host is extracted and HTTP port 7474 is used.
  • "username" — required
  • "password" — required
  • "database" — database name (default "neo4j")

Provider Interface Methods

Mem0.generate_response — Function
generate_response(llm, messages; response_format=nothing, tools=nothing, tool_choice="auto")

Generate a response from the LLM. Must be implemented by all providers.

Arguments

  • messages: Vector of message dicts with "role" and "content" keys
  • response_format: Optional format constraint (e.g., "json")
  • tools: Optional vector of tool definitions for function calling
  • tool_choice: Tool calling mode ("auto", "required", "none")

Returns

An LLMResponse with content and optional tool_calls.

Mem0.embed — Function
embed(embedder, text; memory_action=nothing)

Generate an embedding vector for the given text.

Arguments

  • text: The text to embed
  • memory_action: Optional hint ("add", "search", "update") for action-aware embeddings

Returns

A Vector{Float64} embedding.
